# GUIÓN: XML Y SINTAXIS

## FICHA TÉCNICA
- **Duración**: 2 minutos
- **Tono**: Preciso y didáctico.

---

### 00:00 - INTRODUCCIÓN

**[LOCUTOR]**:
SVG no es un formato de imagen binaria como JPG. Es texto.
Si abres un archivo SVG con el Bloc de Notas, puedes leerlo. Y el idioma en el que está escrito se llama XML.

---

### 00:30 - EL PROFESOR ESTRICTO

**[LOCUTOR]**:
Seguro conoces HTML. HTML es permisivo. Si se te olvida cerrar una etiqueta, el navegador intenta arreglarlo.
XML no. XML es estricto.
Si olvidas una comilla en un atributo, todo el gráfico se rompe.
Si escribes "Circle" con mayúscula en vez de minúscula, no aparece nada.

---

### 01:00 - REGLAS DE ORO

**[LOCUTOR]**:
Tres reglas para sobrevivir:
1. Cierra siempre tus etiquetas.
2. Pon comillas a todos los valores.
3. Respeta las mayúsculas y minúsculas.
SVG es código. Trátalo con el mismo respeto que tratas a JavaScript.

---

### 01:30 - CIERRE

**[LOCUTOR]**:
Entender esto te ahorrará horas de frustración cuando u gráfico "simplemente no aparezca".
La mayoría de las veces, es solo un error de sintaxis XML.
Ahora que sabemos escribir, aprendamos dónde dibujar.
